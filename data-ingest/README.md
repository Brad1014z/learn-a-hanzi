# :data-ingest — the dataset pipeline (spec 02, milestone M2)

A plain JVM tool, never shipped in the APK. It turns pinned open datasets + reviewed
generated content into the bundled SQLite the app reads.

## The three steps

| Step | Command | Network? | When |
|---|---|---|---|
| 1. Fetch raw sources | `./gradlew :data-ingest:downloadData` | yes (pinned URLs) | once per machine, or when re-pinning |
| 2. Build the dataset | `./gradlew :data-ingest:run` | **no** — deterministic | whenever data/pinned or the schema changes |
| 3. Generate TTS clips | `GOOGLE_TTS_API_KEY=… ./gradlew :data-ingest:generateAudio` | yes (billable) | once per dataset change |

- Step 1 verifies every file against `data/sources.lock` (sha256) and **fails loudly on
  upstream drift** — re-pinning is a deliberate, reviewed commit.
- Step 2 reads `data/raw/` + `data/pinned/`, validates everything (HSK 1 characters with
  incomplete data **fail the build**), and writes
  `app/src/main/assets/databases/hanzi_v1.sqlite` (schema taken from the KSP-exported
  Room schema JSON, so `createFromAsset` can never see drift) plus
  `data/ingest-report.md` (counts, warnings, attribution manifest).
  Same inputs → same bytes.
- Step 3 synthesizes one MP3 per character/word/sentence into
  `app/src/main/assets/audio/zh-Hans/` with a manifest; the app's
  `PregenAudioSpeechService` plays them and falls back to device TTS for anything
  uncovered. **No key, no clips — the app still works** (device TTS).

## Content that humans review (the LLM path — spec 00/02 amendment)

- `data/pinned/sentences-hsk1.json` — example sentences generated by the pinned model
  (`claude-fable-5`, prompt in `prompts/sentence-generation.md`), vocabulary-constrained
  to the HSK 1 set. **Review = PR review**: edit/delete lines before merging, or set
  `approved: false`. The ingest tool hard-fails on any off-list character.
- `data/pinned/worlds-hsk1.tsv` — the hand-curated world grouping (names are
  placeholders; naming belongs to the co-designer, spec 11).
- `data/pinned/hsk1-words.json` — the pinned HSK 1 (2.0) word list (150 words,
  178 unique characters).
